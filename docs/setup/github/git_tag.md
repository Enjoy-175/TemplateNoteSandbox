# タグの作成

## 基本ルール
タグは特定のコミットに付けます。
コミットを指定しない場合、現在のブランチの最新コミットに付きます。

## 注釈付きタグ（推奨）
```bash
git tag -a v1.4 -m 'version 1.4のリリース'
```

### 特徴
- タグ作成者の名前とメールアドレスが記録されます。
- タグ作成日時が記録されます。
- コメントを付けられます。
- `-m`を省略するとエディタが開きます。

## 軽量タグ
```bash
git tag v1.4
```

### 特徴
- コミットへのポインタだけが作成されます。
- 作成者情報や日時は記録されません。
- 一時的なマーカーに使います。

## 過去のコミットにタグを付ける
```bash
git tag -a v1.2 -m 'version 1.2' 9fceb02
```

コミットハッシュを指定すれば、過去のコミットにもタグを付けられます。
付け忘れても後から追加できます。

# タグの共有

## リモートへプッシュ
```bash
git push origin v1.5
```

タグはローカルで作成しただけではリモートに反映されません。
必ず`push`コマンドでリモートに送信してください。

## 全タグを一括プッシュ
```bash
git push origin --tags
```

複数のタグをまとめて送信できます。

# タグの確認

## タグ一覧を表示
```bash
git tag
```

すべてのタグがアルファベット順に表示されます。

## パターンで検索
```bash
git tag -l 'v1.4.*'
```

ワイルドカードを使って特定パターンのタグだけを表示できます。

## タグの詳細を表示
```bash
git show v1.4
```

### 軽量タグの場合の表示内容
- コミットハッシュ
- Author（コミット作成者）
- Date（コミット日時）
- コミットメッセージ

### 注釈付きタグの場合の追加表示内容
- Tagger（タグ作成者）
- Tag Date（タグ作成日時）
- 注釈メッセージ

# タグの削除

## ローカルのタグを削除
```bash
git tag -d v1.4
```

まだプッシュしていないタグはこのコマンドで削除できます。

## リモートのタグを削除
```bash
git push origin :refs/tags/v1.4
```

または

```bash
git push origin --delete v1.4
```

### 重要な注意事項
**既にプッシュしたタグは原則削除しないでください。**

理由：
- 他の開発者がそのタグを使っている可能性があります。
- リリース済みのバージョンタグを削除すると混乱を招きます。

どうしても削除が必要な場合は、チーム全体に事前通知してください。
