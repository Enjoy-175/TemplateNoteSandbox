# CAT_004 カテゴリ更新API
- Method: PATCH
- Path: /api/categories/{id}
- 認証: 要（セッション必須）

## Input
- sessionUserId（UUID, 必須, セッションから取得, 例: "5b8e7e4a-4f21-4d0e-b72f-8d5a2d5c9f3b"）
- id（整数, 必須, 正の整数, 例: 12）
- catName（文字列, 必須, 1〜16文字, 空白のみ不可, 例: "仕事"）
- description（文字列, 任意, 0〜50文字, 例: "業務メモ"）

## Validation Rule
- Rule V001: id 必須チェック
  - 対象: id
  - 条件: null
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "入力値が不正です。"

- Rule V002: id 正数チェック
  - 対象: id
  - 条件: id <= 0
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "入力値が不正です。"

- Rule V003: カテゴリ名の必須チェック
  - 対象: catName
  - 条件: null または 空文字 または 空白のみ
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "カテゴリ名は必須です。"

- Rule V004: カテゴリ名の文字数チェック
  - 対象: catName
  - 条件: 文字数 > 16
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "カテゴリ名は16文字以内で入力してください。"

- Rule V005: 説明の文字数チェック
  - 対象: description
  - 条件: 文字数 > 50
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "説明は50文字以内で入力してください。"

- Rule V006: 正規化チェック
  - 対象: catName, description
  - 条件: trim 後に catName が空白、または catName/description が上限超過
  - 判定: NG
  - エラーコード: E-400-VALIDATION
  - メッセージ: "入力値が不正です。"

### チェック順序
1. V001 → NGなら終了
2. V002 → NGなら終了
3. V003 → NGなら終了
4. V004 → NGなら終了
5. V005 → NGなら終了
6. V006 → NGなら終了
7. 全てOKなら次の処理へ

## Process
- Step 1: セッション認証
  - SessionUserInterceptor でセッション確認
  - NG: E-401-UNAUTHORIZED を返して終了

- Step 2: 入力バリデーション
  - Validation Rule を順に適用
  - NG: E-400-VALIDATION を返して終了

- Step 3: 入力正規化
  - catName/description を trim
  - catName が空白、または上限超過の場合は E-400-VALIDATION

- Step 4: 既存カテゴリ取得
  - id でカテゴリを取得
  - 条件: カテゴリが存在する
  - NG: E-404-CATEGORY-NOT-FOUND を返して終了

- Step 5: 所有者チェック
  - 条件: 取得したカテゴリの user_id == sessionUserId
  - NG: E-403-CATEGORY-FORBIDDEN を返して終了

- Step 6: 重複チェック
  - user_id + name の一意制約を確認（自身の id を除外）
  - NG: E-409-CATEGORY-DUPLICATE を返して終了

- Step 7: 更新
  - categories を更新
  - トランザクション開始
  - 失敗: ロールバックし E-500-DB

- Step 8: レスポンス生成
  - 更新後のカテゴリを返却

## Output
### パターン1: 成功
- 型: JSON
- HTTP: 200
- 例:
  {
    "id": 12,
    "catName": "仕事",
    "description": "業務メモ"
  }

### パターン2: Validation Error（例: V001）
- 型: JSON
- HTTP: 400
- 例:
  {
    "code": "E-400-VALIDATION",
    "message": "入力値が不正です。",
    "details": [
      { "field": "id", "message": "入力値が不正です。" }
    ],
    "operation": "update",
    "categoryId": 12
  }

### パターン3: 業務エラー（存在しない）
- 型: JSON
- HTTP: 404
- 例:
  {
    "code": "E-404-CATEGORY-NOT-FOUND",
    "message": "カテゴリが存在しません。",
    "details": null,
    "operation": "update",
    "categoryId": 12
  }

### パターン4: 業務エラー（権限なし）
- 型: JSON
- HTTP: 403
- 例:
  {
    "code": "E-403-CATEGORY-FORBIDDEN",
    "message": "他のユーザーのカテゴリは操作できません。",
    "details": null,
    "operation": "update",
    "categoryId": 12
  }

### パターン5: 業務エラー（重複）
- 型: JSON
- HTTP: 409
- 例:
  {
    "code": "E-409-CATEGORY-DUPLICATE",
    "message": "同じカテゴリ名が既に存在します。",
    "details": null,
    "operation": "update",
    "categoryId": 12
  }

### パターン6: 認証エラー
- 型: JSON
- HTTP: 401
- 例:
  {
    "code": "E-401-UNAUTHORIZED",
    "message": "セッションユーザーが見つかりません。",
    "details": null,
    "operation": "update",
    "categoryId": 12
  }

### パターン7: システムエラー
- 型: JSON
- HTTP: 500
- 例:
  {
    "code": "E-500-DB",
    "message": "システムエラーが発生しました。",
    "details": null,
    "operation": "update",
    "categoryId": 12
  }

## Error 定義
- E-400-VALIDATION
  - 発生条件: id/catName/description のバリデーションNGまたは正規化失敗
  - Input例: id = 0
  - 検出Step: Step 2 / Step 3
  - Output例: Output パターン2

- E-404-CATEGORY-NOT-FOUND
  - 発生条件: id に一致するカテゴリが存在しない
  - Input例: id = 9999
  - 検出Step: Step 4
  - Output例: Output パターン3

- E-403-CATEGORY-FORBIDDEN
  - 発生条件: カテゴリの user_id が sessionUserId と一致しない
  - Input例: id = 12（他ユーザーのカテゴリ）
  - 検出Step: Step 5
  - Output例: Output パターン4

- E-409-CATEGORY-DUPLICATE
  - 発生条件: user_id + name の一意制約違反
  - Input例: catName = "仕事"（同名が既存）
  - 検出Step: Step 6
  - Output例: Output パターン5

- E-401-UNAUTHORIZED
  - 発生条件: セッション未確立または期限切れ
  - 検出Step: Step 1
  - Output例: Output パターン6

- E-500-DB
  - 発生条件: DBアクセス例外
  - 検出Step: Step 7
  - Output例: Output パターン7

- E-500-UNEXPECTED
  - 発生条件: 予期しない例外
  - 検出Step: 全Step
  - Output例:
    {
      "code": "E-500-UNEXPECTED",
      "message": "予期しないエラーが発生しました。",
      "details": null,
      "operation": "update",
      "categoryId": 12
    }
