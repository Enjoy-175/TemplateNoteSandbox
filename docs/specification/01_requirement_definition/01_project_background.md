# プロジェクト背景

---

## どんな課題を解決するのか

### 自己流の学習を促進する

知見を記録するテンプレートを自分で設計し、試行しながら磨く。
誰かに押し付けられない自由さと自作への愛着が、学習モチベーション向上につながる。

### 記録の検索性を独自で用意できる

用途や目的に沿って自由に仕分けできる。
アプリで固定化されたカテゴリではなく、自分の粒度で複数のインデックスを用意できる。

### 想起しやすい形に記録を考える仕組み

- 自作のテンプレートや仕分け情報を使い、自分で検索導線や想起方法を試行錯誤する
  - テンプレートや仕分けに上限を設け、上限内で工夫を促す
    - ユーザーごとに用途を分け、複数アカウントを作成できるようにする
  - 限られた文字数の中で必要な情報だけを選び記録する
  - 取捨選別と整理の過程を通じて、記憶の定着と理解の深化につながる
  - 日々の記録を抽象化し、知りたい内容の欲求を明確にする

#### 補足: 振り返るための仕組みについて

「振り返り」は、当時の理解を失ってしまった未来の自分に託すダイイングメッセージと捉える。
まず、未来の自分が振り返りやすい形で記録を残せるかが重要になる。

- 単なる文章の羅列ではなく、文章量を最小限に留める
- 本人が納得できる構造で記録する

そして、詳細を見る前にいったん思い出そうとする過程を設ける。
これにより記憶の定着と理解の深化を促す。
- 詳細を見る前にいったん思い出す間を置き、思い出せない点や当時の疑問・気づきを整理してから詳細を確認する

## ゴール

- 自分好みのフォーマットをテンプレートとして保存し、目的・用途に応じて柔軟に変更できる
- 振り返りやすく検索できるよう、ユーザー独自のインデックスを用意できる
- 一覧画面では記録詳細を伏せ、詳細画面を開くまでは記載した情報を開示しない
  - 一覧画面は日付やカテゴリ分けなどのメタ情報に限定する
    - 一覧画面では「何を意図して」書いたかを想起させる
  - 単語帳の答えをめくるように、詳細画面で答え合わせするイメージ
- 記録と振り返りをPC上で完結させ、蓄積を学びに直結させる

## 現状困っていること

- 既存ツールが自分の記録スタイルに合わない
  - メモ系で管理すると振り返りが閲覧中心になり、想起を促しにくい
  - 一方でメモや既存の学習アプリは検索と再発見に時間がかかり、振り返りが弱い

### ペルソナ体験: 資格試験

- 教科ごと・年単位では検索できるが、解きたい問題をピンポイントで探すのに手間がかかる
  - 例: 「ネットワークのOSI参照モデルに関する問題だけ解きたい」→「ネットワーク」のカテゴリから総当たりで探す必要がある
- 過去問を解いた後に、「想起」を促す形で自分なりにまとめておきたい
  - 自分なら「問題」「答え」「気づき」の3軸で「思考の足跡」を簡潔にまとめたい
  - 本格的に紙ベースで勉強する時間があまり取れない
    - メモは独自フォーマットを確保しやすいが、コピペできず、工夫しないと隠したい場所が見えてしまう
    - 単語帳は「想起」を促しやすいが、構成を直しにくく蓄積すると探しにくい
  - PC上で完結させたい（記事やChatGPTの要旨を整理し、参照URLで関連情報をまとめたい）
- 「考える」「想起」を促す振り返り体験が不足している
  - 既存の学習アプリは間違えた問題の振り返りが中心（同じ問題が出るとは限らない）
    - 記録しながら理解を深める体験が不足している
    - **間違えた問題から「何に活かすのか」「別の条件ならどうなるか」などの独自の発想**を加えたい人への配慮が不足している

### ペルソナ体験: 読書記録

- 読書で得た気づきをまとめる際は決まったメモの仕方があるが、メモが数冊に分かれて検索性がない
  - フォーマットにこだわるとメモを贅沢に使うこともあり、数冊を超えると探しにくい
    - 1ヶ月で1冊以上消費するため、かさばって振り返りにくい
  - 一方でケチったメモ運用ではフォーマットが崩れやすく、振り返りにくい
- 決まったテンプレートを毎回用意するのが面倒
  - 「メモの魔力」＋αに沿い、「ファクト」「思考」「抽象化」「転用」の4軸でまとめる
  - アプリベースでは検索性は高いが、
    - 結局テンプレートがなく、毎回用意する必要がある
    - フォーマットが固定化されがちで、柔軟に変更できず愛着が湧きにくい

## ゴールとのギャップ

現状の記録管理は以下のとおり。

- 既存のメモ系アプリは検索性と閲覧性に優れるが、想起を促す振り返り体験が不足している
- 問題アプリとのギャップは以下
  - 想起を促す仕組みはあるが、検索性に乏しく、記録フォーマットの柔軟性も低い
  - ユーザー自身の気づきなど独自の知見とセットで記録する仕組みが乏しい
- 振り返りやすさは紙ベースが優れるが、PC上で完結しない。ネット記事の文章はコピペで再利用しにくい
